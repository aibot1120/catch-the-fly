# 捉烏蠅 (Catch The Fly) - 遊戲設計文檔

## 1. 遊戲概述

**項目名：** 捉烏蠅  
**類型：** 多人卡牌遊戲（2-4人本地 multiplayer）  
**平台：** Steam（PC）/ Android / iOS  
**引擎：** Unity Tuanjie  
**核心理念：** 玩家輪流出牌或抽牌，透過「點數匹配」機制造成對手「出銃」或自己「自摸」來得分

---

## 2. 遊戲規則（完整版）

### 2.1 基本設定
- 使用一副**兩人對戰用特殊撲克牌**（共48張，見卡牌設計章節）
- **4名玩家**（東南西北位）
- **莊家**由系統隨機決定，每局結束後**連莊**
- 起始分：**0分**，遊戲持續到有玩家達到**500分**或以上，該玩家獲勝

### 2.2 卡牌類型
| 類型 | 點數 | 張數 | 說明 |
|------|------|------|------|
| 數字牌 | 1-9 | 每點數4張，共36張 | 核心出牌 |
| 功能牌 | 0 | 8張 | 特殊效果 |
| 烏蠅牌 | 0 | 4張 | 萬用牌，可配任何數字 |

**總計：48張**

### 2.3 遊戲流程
1. **發牌：** 系統隨機發給每位玩家**7張**手牌，莊家出第一張
2. **回合制：** 從莊家開始順時針，每位玩家輪流：
   - **出牌**：打出一張符合規則的牌
   - **摸牌**：若無法出牌，必須抽一張
3. **結束條件**：任一玩家手牌歸零 → 該局結束，結算分數

### 2.4 出牌規則
玩家可以出的牌：
- **數字匹配**：出的牌點數 = 桌面公共點數
- **烏蠅牌**：可當任意點數使用
- **功能牌**：打出的點數等於功能牌所示

**烏蠅牌使用限制：** 持有烏蠅牌時，**強制出烏蠅牌**（即使有其他合規出牌）

### 2.5 結算規則

#### 2.5.1 自摸（勝出方式）
當任一玩家**抽到一張牌後**，導致：
- 手上所有牌（7張）的點數總和 = 公共點數
- 或手上**4張同點數**湊成特定組合

→ 該玩家宣佈「**自摸**」，立即勝出此局

#### 2.5.2 出銃（觸發對手勝出）
當任一玩家**出牌後**：
- 下一家檢查：出的牌點數是否等於自己**任意一張手牌**的點數
- 若等於 → 該玩家**出銃**，對手勝出

#### 2.5.3 計分方式
| 勝出方式 | 計算公式 |
|----------|----------|
| 自摸勝出 | 輸家手牌數 × 2 |
| 出銃勝出 | 輸家手牌數 × 2 |
| 自摸四張（4張同點）| 輸家手牌數 × 4 |
| 連莊自摸 | 分數 × 1.5（莊家）|

#### 2.5.4 計分示例
```
場面公共點數 = 15
玩家A手牌：[3, 5, 7, 7, 8, 12, 15] → 15=15 自摸！贏9×2=18分
玩家B出牌：出一張[6]，玩家C手牌：[6, 4, 4, 9] → 6=6 出銃！玩家C勝出
```

## 3. 卡牌詳細設計

### 3.1 數字牌（1-9）
- 每點數 **4張**（對應4名玩家）
- 牌面顯示：點數 + 花色背景

### 3.2 功能牌（8張）
| 功能 | 點數 | 效果 |
|------|------|------|
| 反轉 | 0 | 逆時針出牌 |
| 跳過 | 0 | 跳過下一家 |
| +2 | 0 | 下家摸2張 |
| 轉向+2 | 0 | 指定下家摸2張並逆轉方向 |

### 3.3 烏蠅牌（4張）
- **萬用**：可當任意 1-9 點使用
- **強制**：有烏蠅必出烏蠅
- **計分**：視為所代替的點數

## 4. UI/UX 設計

### 4.1 主介面
- 4人座位分佈（圓形/十字）
- 中央公共區域（公共點數顯示）
- 每位玩家手牌區
- 分數板（實時）

### 4.2 卡牌視覺
- 正面：數字 + 背景圖案
- 背面：統一的「烏蠅」Logo 圖案
- 動畫：出牌/抽牌/結算動畫

## 5. 技術架構

### 5.1 場景結構
- `MenuScene` - 主選單
- `GameScene` - 遊戲主場景
- `ResultScene` - 結算畫面

### 5.2 核心類別
| 類別 | 職責 |
|------|------|
| `GameManager` | 遊戲流程控制 |
| `Player` | 玩家狀態/手牌管理 |
| `CardDeck` | 發牌/抽牌邏輯 |
| `Card` | 卡牌數據（點數/類型） |
| `UIManager` | UI 更新 |
| `AudioManager` | 音效管理 |
